## Question 2: Explain the difference between stractured data and unstractured data and give two examples in each.
**Stractured data** is data that is organized in a fixed format, usually in rows and columns, making it easy to searchand analyze.
For example:
1. A spreadsheet of student names and grades.
2. A bank transaction table with dates, amounts and account numbers.
**Unstractured data** is data that has no predefined format or organization making it harder to process and analyze directly.
For example: 
1. An audio file.
2. A video file of a photo.


## Question 3: What is a Schema? Explain using a simple example.
A schema is a blurprint or stracture that definens how data is organized. It tells you what fields exists and what type of data goes in them, and how they relate to each other.
For example: If you have a student record, the schema would define that it must have:
1. name which is a text.
2. age which is a number.
3. course which is a text.
just like a form has blank fields you must fill in, the schema defines what those fields are before any data is entered.


## Bonus question: Explain the difference between a list and a dictionary.
A **list** is an ordered collection of items accessed by their position (index). You use it when order matters and items are of the same kind.
Example: a list of programming languages, you access them by position like languages[0]
A **dictionary** is a collection of key-value pairs where each item has a label (key). You use it when you want to describe something with named properties.
Example: a student record, you access values by name like student["name"]
In short, a list is like a numbered queue, a dictionary is like a labeled form.
